A: How to install RemoteTracker?
  You have to download the RemoteTracker.CAB file, copy to your mobile device and using the File Explorer (or any other file/folder manager) you have to click the file. The installation procedure will start automatically. Remember to install in Main Memory.
Top

A: How to uninstall RemoteTracker?
  The uninstallation procedure is equal than others. The only difference is if you have a password defined. In this case you will be asked to type your password. This procedure can protect you against non authorized people to uninstall this software. After type the password, you will be asked if you want to keep your configurations or not. You will not find the name RemoteTracker in the list of installed programs. You must look for JVSoftware Configurations to uninstall.
Top

A: How to upgrade RemoteTracker?
  You have to download the new RemoteTracker.CAB file, copy to your mobile device and using the File Explorer (or any other file/folder manager) you have to click the file. The installation procedure will start automatically. The uninstallation procedure will be used and if you have a password you will have to type it. When you will be asked if you want to keep your configuration, click yes. Once again, remember to install in Main Memory.
Top

A: I got an error or it's not working on my device. How can I get support?
   RemoteTracker is a free software without any warranty. Please read the license page to see details. But, you can ask for help if you follow the steps below:

- Read the manual to be sure you know how RemoteTracker works;
- Read this FAQ to be sure your answer is not documented already;
- If you didn't find your answer in Manual and FAQ, you will need to send debug informations to be analyzed by the author. Using the Config application, check the Debug Mode option (Menu/Options/Debug Mode);
- With debug mode checked, try again. Three files can be create in \Temp folder: smslauncher.debug.txt, config.debug.txt and rtdebug.txt;
- Copy that files and send them to the author with a detailed explanation. Please be specific about your needs and tell everything about your device and environment.

Once again, there are no guarantee you will receive an answer because there is a lot of messages being analyzed every day.
Top

A: How to send commands?
  It's simple. You have to send a SMS, just like you send when you want to talk to a friend. The difference is: the message must have a pre defined format, and started with rt# (or rt2# for HTC HD2 users) followed by the command and parameters. For example, if you want to know the current coordinates, you have to send this sms: rt#gp where rt# is the header and gp is the command. Please take a look in this link to see detailed informations.
Top

A: If I lose my device, can everyone see if RemoteTracer is installed?
  By default yes. But you can, after make the full configuration, use the Hide feature using the Option/Hide RemoteTracker menu option. Who find your device must be a power user at lease to know if RemoteTracker is installed or not.

A: If the theft changed the SIM card can I still control my device?
  Yes! RemoteTracker has a security system started everytime your device is turned on. This system verify if the SIM card is in the list of known SIM cards (up to four). If not you will receive a message alerting you about the SIM card change with useful informations, like the new IMSI, number and owner name.
Top

A: If the theft hardreset my device, will I lose RemoteTracker?
  Unfortunatelly yes. But there are hope. If you are a power user, you can make your own custom ROM with RemoteTracker inside. Use the RTRegCreator utility to create the rt.dll file with your configurations inside and cook it with RemoteTracker.

A: What the TopSecret application is?

A: I found a bug. How can I send it to be fixed?
  To be more easy to fix the bug, set the Debug Options ON and reproduce the problem again. Three files can be created in \Temp folder: configdebug.txt, rtdebug.txt and smslauncher.debug.txt. Send that files to the author or to the forum. You can also take screen shots.

A: The RemoteTracker works on CDMA devices?
  Yes and No. RemoteTracker was designed for GSM devices, but you can use it in CDMA devices. Some features with SIM card interactions will not work.

A: I have more than one SIM card. RemoteTracker will alert me when I change them?
  You can configure up to four known SIM cards and you can change them any time you need. The alert will be sent only if someone change for a unkown SIM card.

A: My device don't have a built in GPS receiver. Can I still use it?
  Yes. There are a lot of commands without GPS interaction and you can use them without restriction. For devices without GPS RemoteTracker uses the free OpenCellID web services to retrieve the position based on the tower location your device is connected. Please take a look in OpenCellClient project to learn more about how to add locations to OpenCellID.

A: I forgot my password! Can I retrieve or reset that password?
  There are no way to reset the password. But you can get your password back if you have a secret question/answer defined. Just send the command:

rt#lostpass without password and you will receive a SMS with your secret question. So, send the command:

rt#"secret,[your secret answer]" without password. If your answer is right, you will receive another SMS with your lost password.

If you didn't set the secret question/answer or if you don't remember your secret answer, you will not be able to retrieve your password and you will need a hard reset. If your device has RemoteTracker cocked into your ROM, only reflashing your device you will be able to reinstall RemoteTracker.
Top

A: RemoteTracker stay in memory? It uses very much memory?
  No, RemoteTracker does not stay in memory and because of this RemoteTracker does not use any memory while it's stopped. When you send a command, while processing, some memory will be used and them released.

A: This software intercept every SMS to process commands?
  No! RemoteTracker does not stay in memory and don't intercept any message. When you send a message started with rt#, Windows Mobile will intercept the message and starts RemoteTracker, so, the operating system has a responsibility to start the program.

A: How can I remove my e-mail from the mailing list?
  To unsubscribe from a mailing list:
  • Go the the List Info Page or follow the link provided at the very bottom of any message from the mailing list.
  • Scroll down to the "Remotetracker-list Subscribers" section of the page.
  • Enter your email address in the "To change your subscription (set options like digest and delivery modes, get a reminder of your password, or unsubscribe from Remotetracker-list), enter your subscription email address" field, which is the last text box on the page.
  • Click on the "Edit Options" button.
  • If you don't remember your password for the mailing list (this may or may not be the same as your SourceForge.net user account password), you should click on the "Email My Password To Me" button. You will be able to proceed once the password email has been received.
  • Type your password into the "Password" field in the "Unsubscribing from Remotetracker-list" section of the page.
  • Click on the "Unsubscribe" button.

A: When I send commands to RemoteTracker the SMS is displayed and the command is not processed. What is wrong?
  If you are an HTC HD2 users, please go to that answer. If not, the first thing you must know is: RemoteTracker does not intercept any SMS. Windows Mobile does. If the registry is ok and RT is not started automatically, your Windows Mobile has a problem and we can't do anything about it.
To be sure about the configuration, edit the registry and go to the key:

HKLM\Software\Microsoft\Inbox\Rules

You will see a sub-key named Configuration. Expanding that key you must see the values below:

Command = \Program Files\Configurations\rt.exe
Params = /hide
PostProcess = 1(0X000001)

In MessageFilter sub-key you must see:

CaseSensitive = 0(00000000)
Default = Body
MatchType = 6(0X000006)
Value = rt#

Also, be sure you accepted the ToS. Open the Config application and answer Accept if you see the ToS.

If everything is ok but the commands are not processed, the problem is in your Windows Mobile. Try a soft or hard reset, or try to contact your manufacturer.
Top

A: Why the numbers set as emergency are not receiving messages when I change my SIM card?
  First of all be sure the numbers are correctly configured as emergency numbers. Open the Config application and click Menu/Emergency Numbers. You must type the number as if you would make a call.

After that, be sure the second SIM card is not configured as a known SIM card. Using the Config application again click Menu/SIM Cards. You can use the command GI to see the current IMSI and IMEI numbers.

If everything is ok, change your SIM card to an unkown card and wait a few minutes. After that open the PocketOutlook and click Text Messages/Send Itens. If the last message is the emergency message, RT is working.

If not, SMSLauncher may not be executed automatically. Please edit your registry and go to: HKLM\Init

There are a lot of values here. Search for a value named Launchnn where nn is some number with the string: \Program Files\Configurations\SMSLauncher.exe or other path depending of the language of your Windows Mobile.
If you can't find it, please reinstall RemoteTracker and take a look again. If the problem persist, your Windows Mobile is not able to modify this setting.

If the value is ok, SMSLauncher is started when your device is turned on. In that case, check the Menu/Options/Debug mode option in Config application and try again.
A file named \temp\SMSLauncher.Debug.txt will be created. This file contains information about the problem/success. Please send this file to support.

A: I have an HTC device with HD2 messaging application and I can't do RemoteTracker work. What am I doing wrong?
  HTC HD2 users MUST start the SMS with rt2# instead of rt# header. So, just send commands like that:

rt2#[command]#[number/e-mail]#[password]

It's necessary because HD2 has a very bad code and intercept all SMS ignoring the Windows Mobile SMS Rules API. This is a shame for developers of a company so large and prestigious.
Top

A: I use S2U2 and RemoteTracker doesn't work properly. What can I do?
  It is import that you use the latest version of S2U2 and add rt.exe and SMSLauncher.exe to the exceptions list. To add these files, check the settings for S2U2. Also, make sure S2U2 doesnt shut off the power, and configure RT to use the windows managed GPS.
If you do not do this, the device will shut down before rt.exe has aquired a GPS fix. If this happens, rt.exe will hang and you will not be able to send any more commands. At least, that is what happens on my X1 with WM6.1.
